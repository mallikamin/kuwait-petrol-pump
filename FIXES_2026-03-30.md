# Fixes Applied - 2026-03-30

## Issues Fixed

### ✅ Issue 1: Manual Entry Form - Empty Dropdowns

**Problem**: Shift and nozzle dropdowns were empty/stuck in the manual meter reading form.

**Root Causes**:
1. **No shifts in database** - Database had 0 shifts seeded
2. **Missing API endpoint** - No GET `/api/shifts` endpoint to list all shifts
3. **Type mismatch** - Mobile app types used snake_case but backend returned camelCase
4. **Response structure mismatch** - Mobile app expected array directly, backend wrapped in object

**Fixes Applied**:
1. ✅ **Created shift seeding script** - `apps/backend/seed-shifts.ts`
2. ✅ **Seeded 3 shifts** - Morning (6am-2pm), Afternoon (2pm-10pm), Night (10pm-6am)
3. ✅ **Added GET /api/shifts endpoint**:
   - Service method: `ShiftsService.getAllShifts()`
   - Controller method: `ShiftsController.getAllShifts()`
   - Route: `GET /api/shifts`
4. ✅ **Fixed mobile types** - Updated `Shift` and `Nozzle` interfaces to use camelCase
5. ✅ **Fixed API calls** - Updated MeterReadingFormScreen to access `response.data.shifts` and `response.data.nozzles`
6. ✅ **Updated demo-data.ts** - Added shift creation for future re-seeding

**Verification**:
```bash
# Shifts endpoint now returns 3 shifts:
curl http://localhost:8001/api/shifts -H "Authorization: Bearer <token>"
# Returns: {"shifts":[{Morning},{Afternoon},{Night}]}

# Nozzles endpoint returns 6 nozzles:
curl http://localhost:8001/api/nozzles -H "Authorization: Bearer <token>"
# Returns: {"nozzles":[...6 nozzles...]}
```

### ✅ Issue 2: OCR Returns HTTP 404

**Problem**: Claude Vision API returned "Request failed with status code 404" when processing meter images.

**Root Cause**: Model name `claude-3-opus-20240229` may not have optimal vision support or incorrect API configuration.

**Fix Applied**:
1. ✅ **Updated OCR model** - Changed from `claude-3-opus-20240229` to `claude-3-5-sonnet-20241022` in `apps/mobile/.env`
   - Claude 3.5 Sonnet has better vision capabilities
   - This is a known-good model ID for vision tasks

**File Changed**:
- `apps/mobile/.env` - Line 3: `CLAUDE_MODEL=claude-3-5-sonnet-20241022`

**Testing Required** (User must test on device):
1. Restart Metro: `pnpm --filter @kuwait-petrol-pump/mobile start --host lan --port 8081`
2. Reload app in Expo Go
3. Login as operator
4. Capture meter reading photo
5. Tap "Process" button
6. Verify OCR extracts the meter value successfully
